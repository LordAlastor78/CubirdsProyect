2._ Realizar una especificación semi-formal del TAD Box<T>.

DEFINICION DEL TIPO

Nombre: Box<T>
Declaracion de tipos: Box<T>, T, int, boolean
Caracteristicas:
- Un objeto Box<T> representa una coleccion desordenada con repeticion (multiconjunto).
- No admite elementos null.
- Los objetos del tipo son modificables.

DEFINICION DE LAS OPERACIONES

1) crearCajaVacia() -> Box<T>
Pre: true
Post: devuelve una caja sin objetos.

Necesita: nada.
Modifica: nada (se crea una nueva caja).
Produce: una caja vacía (sin objetos).

2) añadirObjeto(objeto: T)
Pre: objeto != null
Post: añade una ocurrencia de objeto a la caja.
Exc: IllegalArgumentException si objeto == null.

Necesita: objeto != null.
Modifica: la caja actual (se incrementa en una ocurrencia de objeto).
Produce: no devuelve valor.

3) añadirContenido(otraCaja: Box<T>)
Pre: otraCaja != null
Post: añade a la caja actual todas las ocurrencias contenidas en otraCaja.
      El contenido de otraCaja no se modifica.
Exc: IllegalArgumentException si otraCaja == null.

Necesita: otraCaja != null.
Modifica: la caja actual (aumenta su contenido).
Produce: no devuelve valor.

4) eliminarOcurrencia(objeto: T) -> boolean
Pre: objeto != null
Post: si existe alguna ocurrencia de objeto, elimina exactamente una y devuelve true.
      Si no existe ninguna, no cambia la caja y devuelve false.
Exc: IllegalArgumentException si objeto == null.

Necesita: objeto != null.
Modifica: la caja actual (elimina una ocurrencia si existe).
Produce: true si elimina una ocurrencia; false si no existía.

5) recuperarObjeto() -> T
Pre: contarObjetos() > 0
Post: devuelve y elimina un objeto cualquiera de la caja.
Exc: IllegalStateException si la caja está vacía.

Necesita: que la caja no esté vacía.
Modifica: la caja actual (se elimina un objeto).
Produce: un objeto cualquiera de tipo T, distinto de null.

6) contarObjetos() -> int
Pre: true
Post: devuelve el número total de objetos en la caja, contando repeticiones.

Necesita: nada.
Modifica: nada.
Produce: un entero >= 0 con el número total de objetos.


CLASIFICACION DE OPERACIONES (SEGUN TEORIA)

- Constructora: crearCajaVacia
- Modificadoras: añadirObjeto, añadirContenido, eliminarOcurrencia, recuperarObjeto
- Observadora: contarObjetos


INTERFAZ (ESTILO JAVA)

TAD Box<T> {
    Box<T> crearCajaVacia();

    void añadirObjeto(T objeto);

    void añadirContenido(Box<T> otraCaja);

    boolean eliminarOcurrencia(T objeto);

    T recuperarObjeto();

    int contarObjetos();
}


PROPIEDADES SEMI-FORMALES (AXIOMAS)

Sea b = crearCajaVacia():
- contarObjetos(b) = 0

Si x != null:
- tras añadirObjeto(x) en b, contarObjetos(b) aumenta en 1.

Si x != null y eliminarOcurrencia(x) devuelve true:
- contarObjetos(b) disminuye en 1.

Si x != null y eliminarOcurrencia(x) devuelve false:
- contarObjetos(b) no cambia.

Si b2 != null:
- tras añadirContenido(b2), contarObjetos(b) aumenta en contarObjetos(b2).

Si contarObjetos(b) > 0:
- recuperarObjeto() devuelve un elemento no null presente en b y contarObjetos(b) disminuye en 1.
